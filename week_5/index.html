Understanding Node.js

Node.js enables you to build server-side applications, manage files and databases, create APIs, and much more using the same language.

It is an open-source, cross-platform JavaScript runtime environment that allows developers to run JavaScript code outside of a web browser. NPM (Node Package Manager) is the default package manager for Node.js, providing a vast registry of reusable code packages and command-line tools to manage them efficiently.

Node.js is commonly used for server-side and back-end development, including:
- Web servers and real-time applications,
- REST or GraphQL APIs for communication between front-end and back-end,
- Database management (MongoDB, PostgreSQL, MySQL, etc.)
- Automation and server-side scripts,
- Microservices and scalable applications, thanks to its asynchronous, event-driven architecture.

Node.js allows you to use JavaScript for server-side programming, creating APIs, real-time applications, and command-line tools.

Runtime Environment: Node.js uses Google Chrome's high-performance V8 JavaScript engine to compile JavaScript code into machine code, enabling fast execution.

Asynchronous & Non-blocking: It uses an event-driven, non-blocking I/O model, allowing it to handle thousands of concurrent connections efficiently within a single process, without waiting for one task to complete before starting another.

Full-Stack Development: Node.js enables developers to use a single language (JavaScript) for both client-side and server-side development, which streamlines the entire development process.

To run a Node.js program, you typically save your code in a .js file (e.g., app.js) and run it from your terminal using the command node app.js.

Installation

Download the latest official nodejs from the nodejs website.

To check if it's installed run the below code in the terminal which will give the versions if installed on the system.

node -v
npm -v

Modularity

Key components of all robust projects in Node.js is the organization and separation of files. Node.js currently supports two major module systems. Which are stated below:

CommonJS (CJS) - The Classic

This was the original and long-standing standard system in Node.js.

Syntax: Uses the require() function and module.exports.

Execution: Module loading is synchronous (it stops execution until the module is fully loaded).

Usage: It's the default for files with the .js extension unless otherwise specified.

Example:

// exporting module
module.exports = {
add: (a, b) => a * b
};
// importing module
const { add } = require('./math');
console.log(add(5, 5));

ES Modules (ESM) - The Modern Standard

ESM is the official JavaScript standard (introduced in ES6) and is now fully supported and recommended in Node.js for new projects.

Syntax: Uses the import and export keywords.

Execution: Supports asynchronous loading and is generally more efficient.

Usage: Requires either the .mjs extension or setting "type": "module" in your package.json file.

// exporting module
export const subtract = (a, b) => a - b;
// importing module
import { subtract } from './math.js';
console.log(subtract(10, 3));

The Workflow

These functionalities will help you automate and streamline your workflow, saving time and avoiding repetitive tasks.

NPM Scripts

Automate terminal commands directly from your package.json.

For example, you can create a script to start the server:

// package.json
"scripts": {
"start": "node server.js",
"dev": "nodemon server.js"
}

Run the above easily using: npm run start or npm run dev.

Nodemon(The Automatic Refresher)

Nodemon is an essential development-only NPM package. It is crucual because it automatically monitors changes in source files and restarts the Node.js server every time you save.

Installation (as a dev dependency): npm install -D nodemon

.gitignore (Keeping GitHub Clean)

This is an important file for maintaining a secure and clean repository.
This file gives Git instructions on which directories and files to ignore during a commit.
The two most critical items to ignore are the node_modules directory and the .env file.

Security Baseline

For sensitive information that should never be committed to source control (like GitHub), environment variables are used in the .env page. These sensitive data includes API keys, database passwords, and specific configuration settings for different environments.

Example of a .env page:

DB_HOST=localhost
DB_PORT=5432
API_KEY=my_super_secret_key_12345

Understanding NPM

NPM is a crucial part of the Node.js ecosystem, simplifying the process of managing project dependencies.

Package Management: It helps install, share, and manage third-party packages (libraries or modules). The NPM registry is the world's largest single-language code repository, containing over a million packages that add functionality to your applications (e.g., web frameworks like Express.js).

Command-Line Interface (CLI): NPM provides command-line tools to interact with the registry and your project. When you install Node.js from the official website, NPM is automatically installed with it. You can verify the installation by running NPM -v in your terminal.

*package.json*** **File:** This file, typically at the root of a project, acts as the control panel for NPM. It keeps track of project information, the list of all dependencies, and can also define scripts for automating tasks (e.g., NPM run start`).

node_modules Folder: When you install a package, NPM creates a node_modules folder in your project directory to store all the downloaded packages and their associated dependencies.
